# New York Romanian Group (NYRG) | Romania GitHub Community

The open-source home of the New York Romanian Group (NYRG) on GitHub.

This repository hosts the official website for the **New York Romanian Group**. We are a community-driven **Romania GitHub** project providing resources and connections for Romanians in the New York area.

This README is written for collaborators who may be new to GitHub Pages/Jekyll. It focuses on:
- Where to edit content safely
- What files are auto-generated
- How branch preview deployments work
- How the local automation scripts fit together

## Deployment

Production site:
https://newyorkromaniangroup.github.io/

Branch previews:
Any non-main branch is automatically deployed to:

https://newyorkromaniangroup.github.io/<branch-name>/

## Repository structure (high level)

assets/       → CSS, JS, images used by the website  
data/         → generated JSON files used by the frontend  
docs/         → editing guides and technical documentation  
scripts/      → automation scripts for Instagram, gallery, and jobs  
_layouts/     → Jekyll page templates

## Quick edit map

Common changes and the safest file to edit:

- Home page text, Instagram section, featured photos frame: `index.md`
- Mentorship page copy + embedded Google Form: `mentorship.md`
- Jobs board layout (filters/search UI shell): `jobs.md`
  - Job data comes from `data/jobs.json` (generated). Do not hand-edit the JSON unless you know what you are doing.
- Gallery page layout (event cards shell): `gallery.md`
  - Gallery data comes from `data/gallery.json` (generated).

If you are not sure where something lives on the site, open the page’s `.md` file first.
These pages mostly contain HTML with small blocks of collaborator notes.

## Auto-generated data files

Some `data/*.json` files are generated by scripts so non-technical collaborators can update content in Google Drive/Sheets.

- `data/instagram.json`
  - Generated by the Instagram update scripts.
  - The home page reads it in `assets/site.js` to render the latest posts with fallback links.

- `data/gallery.json`
  - Generated from a Google Drive folder (and optionally a sheet of external albums).
  - The gallery page reads it in `assets/site.js` to render event cards.

- `data/jobs.json`
  - Generated from a Google Sheet that includes a Google Form “responses” tab and a “for display” tab.
  - The jobs page reads it in `assets/site.js` to render the job board.

Tip: If you need to make a content change, it is usually easier to edit the source (Google Form/Sheet/Drive)
and then re-run the generator, rather than editing the JSON by hand.

## Technical notes
- Instagram content is **auto-updated** via a script
- Do not manually edit `data/instagram.json` unless you know what you are doing
- Scripts are documented and safe to run locally

## Local automation (optional)

You can update JSON data by running the scripts in `scripts/`.

One maintainer workflow (optional) uses systemd user services and timers:
- Services: `~/.config/systemd/user/`
- Environment variables: `~/.config/nyrg/nyrg.env`

Collaborators do not need systemd to contribute. You can run the generator scripts manually as needed.


## Contributing

- Prefer small, focused PRs (one change at a time).
- Avoid editing generated JSON manually unless you are fixing a one-off emergency.
- If you change CSS or JS, check both:
  - the production root path, and
  - a branch preview path like `/your-branch-name/`
  because `site.baseurl` changes in previews.
